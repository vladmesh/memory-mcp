# memory-mcp

Код memory-MCP сервиса. Часть субстрата секретаря (репо control-panel), но живёт
отдельно: сервис нейтрален к провайдеру и переиспользуем сам по себе.

- Сервис — читатель канона. Пишет в канон только куратор (отдельный репо).
- БД (`memory.db`, sqlite-vec) производная, ребилдится из markdown-канона через `reindex.py`.
- Боевой демон: systemd `memory-mcp`, `127.0.0.1:8077`, модель грузится один раз.
- Менял модель эмбеддингов — останови демон, поправь `MEMORY_MODEL`/`MEMORY_DIM`, прогони `reindex.py`.
